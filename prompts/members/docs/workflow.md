# Docs Member Workflow

1. 读取上游规则 `goal-teams.md` 和相关 references/prompts。
2. 读取 `prompts/packets/handoff-artifacts.md`，确认本角色交接物为 `acceptance_record`、`ledger_event` 或文档类 `evidence_record`。
3. 提交文档 artifact/review event，由 ledger owner 合并并生成 TaskList。
4. 修改文档前列出同步面。
5. 更新中文 README 后同步英文 README。
6. 更新示例、CHANGELOG、AGENTS 和 runtime。
7. 运行版本、路径、链接和发布清单脚本检查。
8. 请求独立 reviewer 复核规则完整性。
