# Goal Teams V2.34 完成说明

Goal Teams V2.34 增加了可恢复的执行控制层，并以 append-only ledger、Evidence、Review 与 Completion Audit 作为验收事实源。

## 已交付能力

- 固定使用 `Gather → Reason → Act → Verify → Repeat`，并在有副作用动作前持久化阶段、轮次、意图和约束。
- 使用 `feature_list.json`、`progress.md`、`contract.md`、`log.md` 保存可恢复状态；混合 revision、摘要漂移或不可证明的中断会失败关闭。
- 实现 contract-first 门禁：冻结可测试断言后，依次完成独立合同评审、架构验收、开发环境检查与实现验证。
- 第 9 轮只允许重置预授权的 disposable candidate，并将其隔离到本地 quarantine；不删除仓库、用户数据或审计证据。
- 第 11 轮执行严格交付门；缺少 required task、current Evidence、独立 Review、全量测试、reset lineage 或 Completion Audit 时不写入 `achieved`。
- 对设计、原创性、工艺和功能性输出可追溯的 0–1 评分；评分不能替代测试或审计。
- 以结构化日志定位 intent 与 judgment 的首个确定性分歧，并约束 Prompt 修订、回归与 holdout 验证。
- 每次 Verify 后从当前阻断图重新计算 moving bottleneck。
- 将公开完成文档净化后原子归档到 `docs/archive/V2.34/<delivery_id>/`；私有过程材料继续保留在本地并排除在发布范围之外。

## 安全边界

V2.34 不声明后台 daemon、跨会话自动运行、生产审批或无限执行能力。破坏性清理、凭证、外部发布和超出既定范围的操作仍需要单独授权。
