# Goal Teams V2.35 发布前摘要

本文档是 V2.35 候选发布的 pre-audit 公开摘要，不是 Completion Audit 结论。

## 候选变更

- 四个只读专家 `goal_security`、`goal_performance`、`goal_refactor`、`goal_sqa` 只提交分析、提案、task patch 与 dispatch request；实现、测试与派发由 Goal Lead 交给独立成员。
- 项目规模 `large|medium|small` 与工作类型 `feature|bugfix` 正交路由；安全、高风险与 UI 覆盖规则不受规模降级。
- unit、TDD、integration、E2E、CLI、API 和 fixture 用例都使用 `input / processing / expected_output / assertions` 结构化契约。
- V2.34 继续作为无 descriptor 时的默认控制面；V2.35 状态、恢复、快照和公开归档需要显式 hash-bound 版本绑定。

## 验证边界

发布候选项必须通过独立 GREEN/全量回归、安全复核、GitHub remote Evidence 和本地安装 Evidence。本文档不宣称这些门已通过。

V2.35 的图外 Completion Audit 尚未运行，仍待远程与本地发布证据闭合后启动。最终 Audit 记录保留在非公开过程包中，不回写本摘要。
