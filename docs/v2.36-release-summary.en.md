# Goal Teams V2.36 Pre-audit Release Summary

This is the public pre-audit summary for the V2.36 release candidate. It is not a Completion Audit conclusion.

## Three-layer version model

- Product version: `V2.36`, used by the Skill package, startup identity, and release documentation.
- General core policy: `V2.5`, identified as `goal-teams-core-v2.5`, for ordinary projects.
- Legacy data schema: `V2.3`, retained for existing ledger, Evidence, Harness, and release-gate data. It is not the current product version.

## Release scope

- The 52 release assertions, iterations 9/11, four-dimension scoring, and public archive now belong to `goal-teams-self-release-v2.36`; they are no longer global invariants for every project.
- `gate_profile` is derived from version, policy profile, task type, execution class, and route facts. Omitting `state_gate_profile` cannot bypass gates, and an explicit value must match the derived result.
- Lite and Standard remain lightweight according to risk, size, and task type; Full and Regulated apply only under explicit escalation conditions. Original UI work uses browser/DOM/geometry/visual Evidence, while replica or reference-driven UI requires a pixel baseline.
- Secret redaction uses one shared implementation covering Authorization/Cookie values, YAML/TOML key-value forms, database URIs, `.netrc`, cloud credentials, and collaboration-tool tokens; non-UTF-8 or control-byte public copies fail closed.
- Evidence uses a protected Git tree snapshot to auto-cover the complete Git change set (tracked modifications/deletions plus non-ignored untracked files), without a caller-selected path list. Final acceptance uses a host-signed route receipt and repository-external persistent challenge state; Audit, Review, and Harness carry the full binding while current Evidence carries a non-circular core binding. The candidate runtime exposes no CLI or Python host-success entry and must return `E_V236_HOST_ADAPTER_REQUIRED`; only a repository-external host may freeze the complete input tree, including TaskList and referenced logs/reports/artifacts, then validate and consume challenges. Caller-selected key/state, caller-supplied `agent_run_id`, and state-free diagnostics remain insufficient.

## Boundaries and verification status

- The self-release Profile applies only to releases of the Goal Teams repository. Ordinary projects do not load its 52 assertions, fixed iterations, or public-archive flow.
- V2.3 paths and schemas remain for compatibility and must not be rewritten as V2.36.
- The graph-external Completion Audit has not run. Release readiness requires version synchronization, focused negative tests, the full `./scripts/check.sh`, release Evidence, and an independent audit; this summary does not claim GA.
