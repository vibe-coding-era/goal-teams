# Goal Teams V2.36 发布前摘要

本文档是 V2.36 候选发布的 pre-audit 公开摘要，不是 Completion Audit 结论。

## 三层版本模型

- 产品版本：`V2.36`，用于 Skill 包、启动语和发布文档。
- 通用核心策略：`V2.5`，机器标识为 `goal-teams-core-v2.5`，用于普通项目。
- legacy 数据 schema：`V2.3`，继续兼容既有 ledger、Evidence、Harness 与 release-gate 数据，不代表当前产品版本。

## 本次范围

- 52 条发布断言、第 9/11 轮、四维评分和公开归档已归入 `goal-teams-self-release-v2.36`，不再作为所有项目的全局不变量。
- `gate_profile` 由版本、policy profile、任务类型、执行等级和 route facts 派生；省略 `state_gate_profile` 不能跳过门禁，显式值也必须与派生结果一致。
- Lite/Standard 根据风险、规模和任务类型保留轻量路径；Full/Regulated 只在明确升级条件下启用。原创 UI 使用 browser/DOM/geometry/visual Evidence，复刻或 reference-driven UI 才强制 pixel baseline。
- secret redaction 使用共享实现，覆盖 Authorization/Cookie、YAML/TOML key-value、数据库 URI、`.netrc`、云凭证和协作工具 token；非 UTF-8/控制字节公开副本 fail closed。
- Evidence 使用受保护 Git tree snapshot 自动覆盖完整 Git 变更集（tracked 修改/删除与 non-ignored untracked），不接受调用方文件清单。最终 acceptance 使用宿主签名 route receipt、仓库外持久 challenge state；Audit/Review/Harness 使用完整 binding，current Evidence 使用非循环 core binding。候选 runtime 的 CLI/Python 均无宿主成功入口，只能返回 `E_V236_HOST_ADAPTER_REQUIRED`；仓库外宿主冻结包含 TaskList 与引用日志/报告/artifact 的完整输入树后才可验证并消费 challenge。自选 key/state、仅自报 `agent_run_id` 或无 state 诊断验证均无效。

## 边界与验证状态

- Self-release Profile 只适用于 Goal Teams 仓库自身发布；普通项目不读取其 52 条断言、固定轮次或公开归档流程。
- V2.3 路径和 schema 保留用于兼容，不应改写为 V2.36。
- 图外 Completion Audit 尚未运行。只有版本同步、定向负向测试、完整 `./scripts/check.sh`、发布 Evidence 和独立审计全部闭合后，才能判断 release-ready；本摘要不声明 GA。
