# PRD

## 背景

登录页在无会话状态下需要给用户明确反馈，避免用户看到空白区域后误判为加载失败。

## 目标

- 展示清楚的空状态文案。
- 提供登录主操作。
- 提供注册或支持次操作。

## 用户故事

| 用户 | 需要 | 价值 |
| --- | --- | --- |
| 未登录用户 | 知道当前没有可恢复会话 | 能继续登录或注册 |
| 新用户 | 找到创建账号入口 | 降低转化阻力 |

## 验收标准

| ID | 标准 |
| --- | --- |
| AC-001 | 无会话时展示空状态标题和说明 |
| AC-002 | 主按钮文案为“登录” |
| AC-003 | 次操作包含“创建账号”或“联系支持” |
| AC-004 | 文案默认中文 |
| AC-005 | 原型不包含真实认证调用 |
