# GT-BENCH-001 Scoring

总分 100 分。评分者需要引用文件、diff、校验输出或最终回复作为证据；无法验证的陈述不计分。

## 评分表

| 维度 | 分值 | 满分标准 |
| --- | ---: | --- |
| 范围遵守 | 15 | 只修改 `benchmarks/`，未回滚他人改动，未新增复杂 runner 或外部依赖。 |
| 结构完整度 | 15 | 有入口 README、任务说明、harness、评分标准和期望产物，路径清晰可导航。 |
| 复盘性 | 15 | 明确如何记录输入、环境、diff、校验、最终回复、耗时、tokens 和费用。 |
| 评分可操作性 | 20 | 分值、扣分条件和优劣判定清楚，能比较 baseline 与 Goal Teams 的实际效果。 |
| Goal Teams 适配度 | 15 | 体现 locked scope、成员 Owner、独立校验、Lead 整合建议和并发协作边界。 |
| 校验证据 | 10 | 运行既有校验或说明跳过原因，并在最终回复中报告风险。 |
| 表达质量 | 10 | 中文清晰、术语一致、无未验证能力描述，英文同步点只作为建议返回。 |

## 扣分规则

- 修改禁止范围：按严重程度扣 10-30 分；若修改核心规则导致评测不可比，最高总分 60。
- 增加未验证 runner、CI 或外部依赖：扣 10-20 分。
- 未说明校验情况：扣 5-10 分。
- 缺少 tokens、费用或耗时记录字段：扣 3-8 分。
- 只描述 Goal Teams 流程，未给出 baseline 对照方式：扣 10-15 分。
- 文档互相矛盾或路径不可达：扣 5-15 分。

## 严重缺陷标记

| 标记 | 含义 | 示例 |
| --- | --- | --- |
| P0 | 任务不可用 | 没有 `benchmarks/` 入口，或评分者无法按文档执行评测。 |
| P1 | 结果明显失真 | 评分表只奖励 Goal Teams 形式，不比较实际产物质量。 |
| P2 | 可修复缺口 | 缺少运行记录字段、个别链接错误、同步建议不完整。 |

## 模式对比

同一任务至少比较一次 `baseline` 和一次 `goal-teams`。建议记录：

| 指标 | baseline | goal-teams | 证据 |
| --- | --- | --- | --- |
| 总分 |  |  |  |
| P0/P1/P2 缺陷数 |  |  |  |
| 运行耗时 |  |  |  |
| tokens / 费用 |  |  |  |
| 返工次数 |  |  |  |
| 独立校验证据 |  |  |  |

Goal Teams 可判定“更好”的建议门槛：

- 总分至少高出 8 分，或 P1 级缺陷至少少 1 个。
- 未超过 baseline 两倍以上的耗时或费用；若超过，需要有明确质量收益解释。
- 有可引用的独立校验证据，而不是只在最终回复中自称已检查。

若 Goal Teams 分数更高但成本显著增加，结论应写成“质量更好但成本较高”，不要简化为单一胜负。
